# ツクルキッズ・シフト作成システム（GAS）

Googleスプレッドシート + Google Apps Script（GAS）で動く、学生スタッフ向けのシフト自動作成・
管理システム。Googleカレンダーの開校日をもとに翌月シフトを自動生成し、Webアプリ（PWA）として
スタッフに公開する。

- **バージョン**: v4.7
- **バックエンド**: GAS（`gas/Code.gs`）+ Googleスプレッドシート
- **フロントエンド**: GASのWebアプリ（`HtmlService`、`index` HTMLテンプレート）

## 主な機能

- **開校日の自動取得**: メインカレンダー＋体験カレンダー（2回/3回体験）から開校日を取得
- **シフト自動生成**: 公平性（勤務回数・時間）を最優先に、ペア条件（セット/禁止）・ワンオペ可否・
  学校時間割・移動時間・希望休を考慮して割り当て
- **プレビュー方式**: 自動生成→プレビュー確認→本反映（または破棄）
- **エリア対応**: 那覇校 / うるま校（土曜のうるま校は固定アサイン）
- **体験期間対応**: 平日2名確保、土曜は ③×3 + ③9:00×1 + ④×2 の増員ルール
- **希望休**: 申請 / 承認・却下（メール通知）/ 集中警告（同日3名以上）
- **シフト交換・急な代打**: 全体発信 → 先着承認 → DB自動反映
- **ヘルプ募集**: 人数不足の枠に、条件を満たすスタッフが自主的に入れる
- **給与計算**: 時間帯別の時間数 × 時給で月次集計
- **ログイン**: スタッフ名＋パスワード / トークン（1年有効）による自動ログイン

## すぐ使いたい人へ（セットアップ）

**[SETUP.md](./SETUP.md) に、コピペで使える状態にするまでの手順をまとめています。**
ポイントは、コードを貼ったあとメニューの **「★ 初回セットアップ」** を1回押すだけで、
必要なシートと初期データ（実スタッフ12名・時間割・勤務エリア・ログインアカウント等）が
自動生成される点です。スタッフ名簿はアップロードされた実データを反映しています。

- 管理者ログイン例：スタッフ名 `管理者` / パスワード `admin1234`
- スタッフログイン例：スタッフ名 `ナカザト` / パスワード `pass1234`
- **本番前に必ず差し替えるもの**：各スタッフのメール・パスワード、`学校時間割` の
  終了時刻（現状すべて仮の 15:00）、`給与設定` の時給、`ADMIN_EMAIL`。

## ファイル構成

```
tsukurukids-shift/
  SETUP.md            # セットアップ手順（手動 / clasp 両対応）
  README.md
  gas/
    Code.gs           # GAS本体（サーバーサイド + Webアプリ + 初回セットアップ）
    index.html        # WebアプリのUI（ログイン/カレンダー/申請/管理画面・PWA対応）
    appsscript.json   # GASマニフェスト（タイムゾーン・Webアプリ設定・スコープ）
    .clasp.json.example  # clasp 用設定のひな形（scriptId を入れてコピーして使う）
  demo/
    demo.html         # ブラウザ内デモ（1画面UI・実12名＋割り当てロジック移植・GAS不要）
```

`demo/demo.html` はブラウザで開くだけで動くプレビュー版（localStorage保存・スプレッドシート/
メール非連携）。本番GAS版とはコードベースが別で、動きを確認する用途のもの。

> `doGet` は `HtmlService.createTemplateFromFile('index')` を参照します。Apps Script 側では
> HTMLファイルの名前が `index`（拡張子なし表示）になるよう、`index.html` の内容をそのまま
> 「ファイル > HTML」で作成した `index` に貼り付けてください（clasp利用時は `index.html` のまま push で可）。
>
> `index.html` は Google Fonts / FullCalendar を CDN から読み込みます。オフラインの
> 完全自己完結ではなく、表示時にインターネット接続が必要です（PWAのService Workerは
> ナビゲーションのみキャッシュ）。

## 取り込み時の修正

リポジトリ取り込みにあたり、動作を妨げる以下を修正しました（元の挙動を変える変更はしていません）。

- **[致命的] `new Daate(...)` → `new Date(...)`**（`fetchOpenDaysByOffset`）
  タイプミスにより、開校日取得・シフト自動生成が起動直後に `ReferenceError` でクラッシュしていた。
- **[コメント] `assignSaturdaySpecial` の ④ 選出人数のコメントを実装（2名）に一致させた**
  （v4.7で ④2名に変更済みだが、コメントが「3名」のままだった）

## デプロイ（clasp を使う場合）

ローカルの `Code.gs` を Apps Script プロジェクトへ反映するには、Googleの公式CLI
[clasp](https://github.com/google/clasp) を使うのが簡単です。

```bash
npm install -g @google/clasp
clasp login

# 既存のスプレッドシート紐付けApps ScriptプロジェクトのスクリプトIDを控えて:
cd application/tsukurukids-shift/gas
# .clasp.json を作成（scriptId はご自身のものに置き換え）
#   { "scriptId": "＜スクリプトID＞", "rootDir": "." }
clasp push
```

> `.clasp.json`（scriptIdを含む）は環境依存のためリポジトリにはコミットしないでください。

手動で反映する場合は、スプレッドシートの「拡張機能 > Apps Script」を開き、`Code.gs` の内容を
貼り付けて保存します。

## 初期セットアップ

1. **スプレッドシートを用意**し、以下のシートを作成（列は本文のとおり）
   - `学生スタッフ情報`（名前 / 学校 / クラス / コース / 土曜可否 / 15:30可否 / 水曜可否 / ワンオペ可否）
   - `学校時間割`（学校 / クラス / コース / 月〜金の終了時刻）
   - `ペア条件専用`（スタッフ / 相手 / 条件[セット|禁止]）
   - `曜日・時間別必要スタッフ数`（曜日 / 時間帯 / 人数）
   - `勤務エリア`（名前 / 月〜土のエリア）
   - `ログインアカウント情報`（スタッフ名 / メール / パスワード）
   - `開校日` / `開校日（翌月）` / `開校日（再来月）`（自動作成される）
   - `完成シフト表_DB` / `希望休申請` / `シフト交換管理` / `給与設定`（必要時に自動作成される）
2. **`Code.gs` を反映**（上記デプロイ手順）
3. **カレンダーIDを確認**
   - メイン開校日: `tsukurukids4f@gmail.com`
   - 体験カレンダー: `TAIKEN_CALENDAR_ID` 定数（`Code.gs` 冒頭）
4. **管理者メールを確認**: `ADMIN_EMAIL` 定数（初期値 `tsukurukids4f@gmail.com`）
5. **Webアプリとしてデプロイ**: デプロイ > 新しいデプロイ > ウェブアプリ
   （実行ユーザー: 自分 / アクセス: 適宜）
6. **トリガー設定**: スプレッドシートのメニュー「⚙️ シフト作成システム >【設定】自動実行トリガーをセット」
   → 毎月1日8時に、今月・翌月・再来月の開校日取得と翌月シフトの自動生成・公表を実行

## 運用フロー

| 日 | 内容 |
|---|---|
| 毎月1日 | 翌月シフト自動生成・公表 |
| 毎月1〜10日 | スタッフ同士のシフト調整（交換）依頼期間 |
| 毎月10日 | 希望休申請の締切 |
| 毎月15日 | シフト確定（以降は変更不可・管理者へ直接相談） |

## 既知の制限・留意点

- **UIはCDN依存**。`index.html` は Google Fonts / FullCalendar を外部CDNから読み込むため、
  表示にはインターネット接続が必要（完全オフライン動作はしない）。
- **同時書き込みの厳密な排他はなし**。シフト交換の承認は「ステータス確認 → 更新」の
  楽観的方式で、極端な同時操作では後勝ちになり得る。
- **メール送信は `MailApp`** に依存（個人Googleアカウントで1日100通程度の上限）。
  送信失敗時も申請・承認処理自体は継続する設計。
- 個人情報（スタッフ名・メール・パスワード等）はスプレッドシート側にのみ保持され、
  本リポジトリにはコードのみを収録している。
