# セットアップ手順（アプリを使える状態にする）

このアプリは Google Apps Script（GAS）+ Googleスプレッドシートで動きます。
以下の手順で「ログインして使える状態」になります。所要 15〜20分。

> ⚠️ この手順は**あなたのGoogleアカウント上での操作**が必要です（外部からは代行できません）。
> 画面の文言どおりに進めれば完了します。

---

## 方法A：手動で貼り付ける（推奨・確実）

### 1. スプレッドシートを作る
1. [Googleスプレッドシート](https://sheets.google.com/) で新規作成
2. 名前を「ツクルキッズ シフト管理」などに変更

### 2. Apps Script を開いてコードを貼る
1. スプレッドシートのメニュー **拡張機能 > Apps Script**
2. 既定の `コード.gs`（`Code.gs`）を全消しして、本リポジトリの
   [`gas/Code.gs`](./gas/Code.gs) の中身を貼り付け
3. 左の「＋」>「HTML」で新規HTMLファイルを作成し、**ファイル名を `index`** にする
   （`.html` は自動で付く。表示名が `index` になっていればOK）
4. その `index` に [`gas/index.html`](./gas/index.html) の中身を貼り付け
5. 💾 保存

### 3. スクリプトプロパティ（任意）
`ACCESS_TOKEN` などは本アプリでは必須ではありません（ログインはスプレッドシートの
`ログインアカウント情報` で管理）。この手順はスキップして構いません。

### 4. 初回セットアップを実行（★重要）
1. スプレッドシートに戻り、**リロード**するとメニューに「⚙️ シフト作成システム」が出る
   （出ない場合は Apps Script で関数 `onOpen` を一度実行 → 承認）
2. **⚙️ シフト作成システム > ★ 初回セットアップ（シート＋サンプルデータ作成）** を実行
3. 初回は権限承認のダイアログが出る → 自分のアカウントで許可
4. 「必要なシートと初期データ」が自動作成される
   （実スタッフ12名・時間割・エリア・ログインアカウント・給与設定 など）

### 5. 開校日を入れてシフトを生成
- カレンダー連携する場合：**メニュー > 1. カレンダーから開校日を取得**
- カレンダーを使わない場合：`開校日（翌月）` シートに直接
  `日付 / 曜日 / イベント名` を手入力（例：`2026/08/04 / 月 / 通常開校`）
- そのあと **メニュー > 2. シフトの自動割り当てを実行**
  （管理画面からは「プレビュー→本反映」も可能）

### 6. Webアプリとして公開
1. Apps Script 右上 **デプロイ > 新しいデプロイ**
2. 種類「ウェブアプリ」
   - 実行するユーザー：**自分**
   - アクセスできるユーザー：**全員**（スタッフに配る場合）
3. 発行された **URL** を控える → スマホ／PCのブラウザで開く

### 7. ログイン確認
- 管理者：スタッフ名 **管理者** / パスワード **admin1234**
- スタッフ例：スタッフ名 **ナカザト** / パスワード **pass1234**

> 🔐 サンプルのパスワードは必ず本番前に変更してください
> （`ログインアカウント情報` シートを編集）。`ADMIN_EMAIL`（`Code.gs` 冒頭）を
> 自分の管理者メールに変えると、そのアカウントが管理者になります。

## 方法B：clasp で push する（コード管理を続けたい人向け）

ローカルのこのリポジトリから直接反映できます。

```bash
npm install -g @google/clasp
clasp login   # ブラウザでGoogle認証

# Apps Scriptプロジェクトを新規作成（スプレッドシートに紐付け）する場合:
#   スプレッドシート > 拡張機能 > Apps Script を一度開き、
#   「プロジェクトの設定」からスクリプトIDを控える

cd application/tsukurukids-shift/gas
cp .clasp.json.example .clasp.json
#  .clasp.json の scriptId を自分のIDに書き換える
clasp push    # Code.gs / index.html / appsscript.json が反映される
```

そのあとは **方法A の手順4以降**（初回セットアップ→開校日→生成→デプロイ）と同じです。

> `.clasp.json` はあなた専用のIDを含むためコミットしないでください（`.gitignore` 済み）。

## うまくいかないとき

| 症状 | 対処 |
|---|---|
| メニューが出ない | Apps Scriptで `onOpen` を実行して権限承認、その後スプレッドシートをリロード |
| ログイン画面でスタッフが出ない | `ログインアカウント情報` シートが空。初回セットアップを実行 |
| 「間に合いません」で誰も割り当たらない | `学校時間割` の終了時刻がシフト開始時刻＋10分より遅い。時刻を見直す |
| カレンダー取得でエラー | `Code.gs` の `mainCalendar` のID／`TAIKEN_CALENDAR_ID` を自分のカレンダーIDに変更 |
| メールが届かない | `MailApp` は1日100通程度の上限。送信失敗しても申請・承認自体は成功する設計 |
